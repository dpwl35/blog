<b>Framework : </b>[Next.js](https://nextjs.org/)  
Gatsby도 고민했었지만 Next.js의 유연성과 Next.js 사용 경험을 쌓기위해 Next.js를 선택했습니다.  

<b>Design  : </b>[Scss](https://sass-lang.com/)  
SCSS는 CSS 전처리기로, 변수, 중첩, 믹스인 등 다양한 기능을 제공하여 CSS를 구조적이고 유지보수하기 쉽게 만들어줍니다. 또한, 개인적으로 깔끔한 클래스명을 선호하고, SCSS의 폴더 구조를 통해 스타일 코드의 가독성과 관리 용이성을 높일 수 있어서 선택했습니다.  

<b>글 : </b>[MDX](https://mdxjs.com/) + [Next-MDX-Remote](https://github.com/hashicorp/next-mdx-remote#react-server-components-rsc--nextjs-app-directory-support)  
Markdown을 기반으로 하여 JSX를 삽입할 수 있어 유연하게 콘텐츠를 작성할 수 있습니다. 이를 통해 블로그 글에 React 컴포넌트를 직접 삽입할 수 있으며, 정적 사이트 생성과의 호환성도 뛰어나 Next.js와 잘 맞아떨어집니다.  

<b>배포 : </b>[Vercel](https://vercel.com/dashboard)  
Vercel은 Next.js의 개발사이기도 하며, Next.js와의 강력한 통합을 자랑합니다. 손쉬운 배포, 자동화된 CI/CD 파이프라인, 서버리스 함수 지원 등 다양한 기능을 제공합니다.

### 구조
```
  public/
    posts: favicon등, 이미지 asset 저장
  src/
    app
    lib 
    components : custom component
    layouts: Header, Footer, Main 등
    posts: mdx 게시글 
    style : scss
```

 게시글 카테고리는 TIL과 Archive로 나눴습니다.
 TIL : 기록 공간 
 Archive : 작업물 공간 


### 기능 개발

#### 1. 블로그 디자인과 scss구조
블로그 디자인을 해야할까 고민하다가 Next.js를 처음 시작했을 때 나오는 메인 페이지 스타일을 살리기로 했습니다. 블로그의 경우 css가 복잡하지 않기 때문에 scss 7-1 패턴 구조에 사용하는 폴더 명을 블로그 규모에 맞게 수정해서 사용합니다.
```
  _base.scss : reset  / Typography 
  _components : components
  _layout : blog layput
  _mixin :  Sass Mixins
  _post : post style (pages)
  _variables : Sass Variables
```
